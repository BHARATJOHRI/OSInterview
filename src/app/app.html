‚ùì What is an Operating System?
An Operating System (OS) is system software that manages computer hardware and software resources. It acts as an interface between user and hardware. The OS handles processes, memory, files, and devices. Examples include Windows, Linux, and macOS.
‚ùì What are the main functions of an OS?
The OS manages processes, memory, files, and input/output devices. It provides security and a user interface. It ensures efficient resource utilization and handles error detection and recovery.
‚ùì What is a Process?
A process is a program in execution. It includes program code, data, and execution state. Each process has its own resources. The OS schedules processes for execution.
‚ùì What is a Thread?
A thread is a lightweight unit of a process. Multiple threads exist within a process and share its resources. Threads improve performance and responsiveness.
‚ùì Difference between Process and Thread?
A process has its own memory space, while threads share memory. Process creation is heavy, thread creation is light. Threads are faster to switch.
‚ùì What is Multitasking?
Multitasking allows multiple programs to run simultaneously. The CPU switches between tasks quickly, improving system efficiency. Common in modern operating systems.
‚ùì What is Multiprocessing?
Multiprocessing uses multiple CPUs or cores to execute tasks in parallel. It increases system performance and is used in servers and high-end systems.
‚ùì What is Multithreading?
Multithreading allows multiple threads within a process to run concurrently. It improves CPU utilization and is common in applications like browsers.
‚ùì What is Scheduling?
Scheduling decides which process executes next. It is managed by the OS scheduler and improves CPU efficiency using different algorithms.
‚ùì What is a Scheduling Algorithm?
Scheduling algorithms determine the execution order of processes. Examples include FCFS, SJF, and Round Robin. They affect waiting time and turnaround time.
‚ùì What is FCFS Scheduling?
First Come First Serve executes processes in arrival order. It is simple to implement but may cause long waiting times and is not efficient in all cases.
‚ùì What is SJF Scheduling?
Shortest Job First selects the process with the shortest burst time. It reduces average waiting time but is difficult to predict. It can be preemptive or non-preemptive.
‚ùì What is Round Robin Scheduling?
Round Robin assigns equal time slices to processes. Each process gets fair CPU time, preventing starvation. Common in time-sharing systems.
‚ùì What is Priority Scheduling?
Processes are scheduled based on priority. Higher priority processes execute first. It can cause starvation of low-priority processes.
‚ùì What is Starvation?
Starvation occurs when a process never gets CPU time. Low-priority processes suffer and fairness is affected. Aging technique helps solve starvation.
‚ùì What is Aging?
Aging gradually increases the priority of waiting processes. It prevents starvation and ensures fairness in priority scheduling.
‚ùì What is Deadlock?
Deadlock occurs when processes wait for each other indefinitely. Resources are never released and the system halts execution.
‚ùì Conditions for Deadlock?
Mutual exclusion, hold and wait, no preemption, and circular wait. All four must occur simultaneously for deadlock to happen.
‚ùì What is Deadlock Prevention?
Deadlock prevention avoids one of the four deadlock conditions. It restricts resource allocation to ensure system safety.
‚ùì What is Deadlock Avoidance?
Deadlock avoidance predicts deadlocks before they occur. It uses algorithms like Banker‚Äôs Algorithm and requires prior resource information.
‚ùì What is Banker‚Äôs Algorithm?
Banker‚Äôs Algorithm avoids deadlock by checking whether resource allocation keeps the system in a safe state.
‚ùì What is Memory Management?
Memory management handles allocation and deallocation of memory. It ensures efficient usage and prevents memory leaks.
‚ùì What is Paging?
Paging divides memory into fixed-size pages. It eliminates external fragmentation and uses page tables for address mapping.
‚ùì What is Segmentation?
Segmentation divides memory into logical segments of variable size. It improves program structure but may cause fragmentation.
‚ùì Difference between Paging and Segmentation?
Paging uses fixed-size blocks and avoids external fragmentation. Segmentation uses variable-size blocks and reflects program logic.
‚ùì What is Virtual Memory?
Virtual memory uses disk as extra RAM. It allows large programs to run efficiently and is commonly implemented using paging.
‚ùì What is Thrashing?
Thrashing occurs when the system spends more time swapping pages than executing processes, causing CPU utilization to drop.
‚ùì What is Page Fault?
A page fault occurs when a required page is not present in memory and must be loaded from disk, increasing execution time.
‚ùì What is Fragmentation?
Fragmentation is wasted memory space. Internal fragmentation occurs within blocks, while external fragmentation occurs between blocks.
‚ùì What is a File System?
A file system organizes and manages data storage. It handles files and directories and provides access methods like NTFS and FAT.
‚ùì What is a Directory?
A directory stores file information such as file names and locations, helping organize data and improve accessibility.
‚ùì What is I/O Management?
I/O management handles input and output devices using drivers. It improves efficiency through buffering and caching.
‚ùì What is Buffering?
Buffering temporarily stores data during transfer. It smooths data flow and improves I/O performance.
‚ùì What is Spooling?
Spooling stores data temporarily on disk for slow devices like printers, improving CPU utilization.
‚ùì What is Context Switching?
Context switching saves the state of a process and switches CPU execution to another process. It enables multitasking but adds overhead.
‚ùì What is Kernel?
The kernel is the core of the OS. It controls hardware, system calls, and runs in privileged mode.
‚ùì Types of Kernel?
Types include monolithic, microkernel, and hybrid kernels. Hybrid combines speed and modularity.
‚ùì What is a System Call?
System calls allow user programs to access OS services like file handling and process control. They run in kernel mode.
‚ùì What is User Mode and Kernel Mode?
User mode has limited access, while kernel mode has full system access. This separation ensures system safety.
‚ùì What is Semaphore?
A semaphore is a synchronization tool that controls access to shared resources and prevents race conditions.
‚ùì What is Mutex?
Mutex ensures mutual exclusion by allowing only one thread to access a resource at a time.
‚ùì What is Synchronization?
Synchronization coordinates processes to prevent data inconsistency in concurrent environments.
‚ùì What is Race Condition?
A race condition occurs when multiple processes access shared data simultaneously, producing unpredictable results.
‚ùì What is a Critical Section?
A critical section is a code segment that accesses shared resources. Only one process is allowed at a time.
‚ùì What is Booting?
Booting starts the computer system and loads the OS into memory. It can be cold boot or warm boot.
‚ùì What is Interrupt?
An interrupt signals the CPU to stop the current task and handle urgent events. It can be hardware or software based.
‚ùì What is Polling?
Polling repeatedly checks device status. The CPU actively waits, making it simple but inefficient.
‚ùì What is a Time Sharing System?
Time sharing divides CPU time among users, improving responsiveness in multi-user systems.
‚ùì What is a Distributed Operating System?
A distributed OS manages multiple systems and appears as a single system, improving reliability and scalability.
‚ùì Why is OS Important?
The OS manages resources efficiently, provides a user interface, ensures security and stability, and is essential for system operation.
